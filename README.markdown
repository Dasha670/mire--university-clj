## Что сделали:
1. Исправления:
   - Обновили в `commands.clj` функцию `execute`. В новой версии добавлена проверка регистра команд через `.toLowerCase` и более информативное сообщение об ошибке для неизвестных команд. Теперь функция предотвращает `NullPointerException` через явную проверку `(if cmd-fn ...)`, которая не дает вызвать `(apply nil args)` при отсутствии команды в `map`'е.
   - Обновили в `server.clj` функцию `main`. Сначала прогружаются комнаты, а только потом присоединяется пользователь. Это исключает возможную ошибку при выполнении команды `move`.
   - Отредактировали вывод `help` и `stats` в терминал.
2. Система монет:
   - В `player.clj` добавили переменную `coins`, которая хранит количество монет. Там же добавлены функции `get-coins` и `add-coins`, которые соответственно получают количество монет и добавляют.
   - В `commands.clj` добавлена функция `coins`, которая отображает пользовалю его баланс.
   - Баланс отображается в инвентаре или при вводе команды `stats`.
3. Система сундуков:
   - В каждой комнате кроме `start` есть сундук. Он открывается с помощью ключей, которые находятся в комнате `closet`. Применяя команду `open-chest`, игрок открывает сундук и ему рандомно начиняются монеты в размере от 5 до 30. После вскрытия исчезают сундук и ключ.
4. Система купли-продажи:
   - При вводе команды `shop` отображается вся информация о магазине.
   - При вводе команды `sell <название_предмета>` игрок продаёт предмет за фиксированную сумму, которая начисляется на его баланс.
   - При вводе команды `buy` покупается эликсир опыта.
5. Система уровней и опыта:
   - При покупке эликсира опыта пользователь получает 10 опыта.
   - Изначально у пользователя 1 уровень. Для достижения каждого из последующих уровней необходимо получить заданное количество опыта, которое отображается при вызове команды `stats`.
### БОТ:
- Он заходит как обычный игрок, и путешествует по комнатам с помощью рандомного направления, которое определяется при заходе в каждую комнату с помощью функции `random_move`.
- При заходе в комнату к игроку, он сообщает ему о своём присутствии.
- Он обновляет наличие необходимых предметов у данной комнаты с помощью функции `spawn-item` описанную в `commands.clj`, а затем выходит в следующую.
